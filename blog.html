Discuss the scope of var, let, and const

VAR

Function Scope: Variables declared with var are function-scoped. They are only accessible within the function where they are defined.

Hoisting: Variables declared with var are hoisted to the top of their containing function or global scope, meaning they can be used before they are declared, but they will be initialized with undefined until their actual declaration.

LET

Block Scope: Variables declared with let are block-scoped, meaning they are accessible only within the block where they are defined. A block can be a function, loop, or any code enclosed in curly braces {}.

Hoisting: Variables declared with let are also hoisted to the top of their containing block like var, but they are not initialized and cannot be used before their declaration.

CONST

Block Scope: Like let, const is also block-scoped.

Mutability: Variables declared with const are meant to be constants, and their value cannot be reassigned after initialization. However, for objects and arrays declared with const, their properties or elements can be modified. The variable itself cannot be reassigned.

Tell us the use cases of null and undefined

null

null is a value that represents the intentional absence of any value. It is typically used to indicate that a variable or property should have no value or that it has been intentionally cleared. It can also be used as a placeholder or initial value that might be assigned a value later in your code. Some functions or methods may return null to indicate that they did not find a meaningful result.

undefined

In JavaScript, variables that have been declared but have not been assigned a value are automatically initialized with undefined. So, undefined is often used as the default value for function parameters and variables. If an object value is accessed that does not exist or an array element is accessed that is beyond the length of the array or if a function does not explicitly return a value, it will return undefined.

What do you mean by REST API?

REST API

A REST API is an application programming interface that adheres to the constraints of REST architectural style and enables interaction with RESTful web services. Interconnected networks make up the web. A web service is a set of open protocols and standards used for exchanging data between client-server applications. Web services that follow the REST architecture are known as RESTful web services.
